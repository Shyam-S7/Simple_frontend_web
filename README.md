# 🥘 Briyani Payaluga - Authentic Restaurant Website

Briyani Payaluga is a sophisticated restaurant frontend application designed to showcase authentic South Indian cuisine. This project focuses on a clean user experience, mobile-first responsiveness, and elegant food presentation.

## 🌟 Overview
This website serves as a digital storefront for **Briyani Payaluga**, featuring a full-service menu, branch locations, and an integrated contact system. It is built with performance and SEO best practices in mind using modern web standards.

## ✨ Key Features
- **Responsive Layout**: Seamlessly transitions between mobile, tablet, and desktop views.
- **Detailed Menu**: Categorized listing of breakfast, signature briyanis, and special meals.
- **Branch Navigator**: Easy access to information regarding different restaurant locations.
- **Interactive UI**: Hover effects, smooth transitions, and premium typography.
- **Contact Integration**: Direct links for phone inquiries and table bookings.

## 🛠️ Technology Stack
- **Structure**: Semantic HTML5 for better SEO and accessibility.
- **Styling**: Vanilla CSS3 with a unified design system and custom properties.
- **Icons**: Font Awesome 6.5.0 integration for a modern look.
- **Typography**: Google Fonts (Inter/Outfit) for a premium feel.

## 📂 Project Structure
```bash
web_hotel/
├── index.html          # Main landing page
├── about.html          # Restaurant history and mission
├── menu.html           # Full culinary offerings and pricing
├── branches.html       # Location details for all outlets
├── contact.html        # Booking and contact form
├── assets/
│   └── css/
│       └── style.css   # Main stylesheet and design tokens
└── scripts/            # Internal utility scripts
```

## ⚙️ Developer Utilities
Two Python-based tools are provided to assist with content management:
- **`apply_unique.py`**: Automatically swaps image placeholders with unique identifiers for realistic prototyping.
- **`download_images.py`**: Mass-downloads external web images, stores them locally in the assets folder, and updates HTML paths automatically.

## 🚀 Getting Started
1. **Clone** the repository to your local machine.
2. **Open** `index.html` in any modern web browser.
3. **Customize** the content in the respective HTML files for your needs.
